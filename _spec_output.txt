1. Overview
The Quoting Engine is the pricing intelligence layer of Myra’s TMS. It accepts a load request (origin, destination, equipment type, weight, pickup date) and returns an instant rate quote. The engine is designed with a cascading data source architecture: it attempts the highest-confidence source first, and falls back through progressively simpler sources until it can produce a quote. Every quote is stored for win/loss tracking, and every completed load feeds back into the engine’s learning dataset.
 
Design Philosophy
The quoting engine should never fail to return a number. Even with zero API integrations configured, it should produce a reasonable estimate from benchmark data. As data sources are added (DAT API, Truckstop API, AI analysis), the confidence and accuracy improve, but the engine always works.
1.1 Feature Location
The quoting engine is a feature within the TMS application. It manifests in three places:
Quote Form Page: A dedicated page/modal in the TMS where the user inputs load details and receives an instant quote with margin breakdown. Accessible from the main navigation and from within the load creation flow.
Inline Quoting: When creating a new load in the TMS, the quoting engine auto-populates the suggested shipper rate and carrier cost estimate based on the load’s parameters.
API Endpoint: POST /api/v1/quotes — returns a structured quote object. This endpoint will eventually power external integrations (shipper portal, email-based quoting).

2. Data Source Cascade — The Failsafe Architecture
The quoting engine uses a cascading priority system. When a quote is requested, the engine attempts data sources in order from highest confidence to lowest. If a source fails (API down, key not configured, no data for the lane), it falls through to the next source. This ensures the engine always returns a quote.
2.1 Source Priority Chain
PRIORITY 1: Myra Historical Data (own completed loads on this lane)
    └─ If 5+ loads on lane in last 90 days → HIGH confidence
    └─ If 1-4 loads → MEDIUM confidence, blend with external
PRIORITY 2: DAT RateView API (if dat_api_key configured)
    └─ Real-time spot + contract rates for the lane
    └─ Cache response in rate_cache table (TTL: 4-8 hours)
    └─ HIGH confidence
PRIORITY 3: Truckstop Rate Analysis API (if truckstop_api_key configured)
    └─ Spot market rate data + rate trending
    └─ Cache response in rate_cache table (TTL: 4-8 hours)
    └─ HIGH confidence
PRIORITY 4: Manually Seeded Rate Cache (rate_cache table)
    └─ Rates manually entered from DAT website lookups
    └─ MEDIUM confidence (degrades with age)
    └─ Flag if older than 7 days
PRIORITY 5: AI-Powered Estimation (if ai_api_key configured)
    └─ Send lane details + any partial data to Claude/OpenAI
    └─ AI analyzes based on market knowledge + provided context
    └─ MEDIUM confidence
PRIORITY 6: Benchmark Formula (always available, no config needed)
    └─ Distance-band rate table × equipment multiplier
    └─ LOW confidence — flag for manual review
 
Critical Rule
The engine NEVER returns an empty quote. If all external sources fail, the benchmark formula always produces a number. Low-confidence quotes are flagged in the TMS with a yellow warning badge prompting the user to verify the rate before committing to the shipper.

3. Data Source Integration Specs
3.1 Source 1: Myra Historical Data
This is the most valuable and most accurate source because it reflects your actual transaction costs on a given lane. Every completed load in Myra’s database is a data point.
Database Query
SELECT
  AVG(carrier_rate) as avg_carrier_cost,
  AVG(shipper_rate) as avg_shipper_rate,
  AVG(margin) as avg_margin,
  MIN(carrier_rate) as min_carrier_cost,
  MAX(carrier_rate) as max_carrier_cost,
  COUNT(*) as load_count,
  MAX(created_at) as most_recent
FROM loads
WHERE status IN ('delivered', 'invoiced', 'paid')
  AND equipment_type = $equipment_type
  AND origin_region = $origin_region
  AND dest_region = $dest_region
  AND created_at > NOW() - INTERVAL '90 days'
ORDER BY created_at DESC
The origin_region and dest_region fields need to be normalized. When a load is created, geocode the addresses and map them to regions. For Ontario, a reasonable region mapping would be metropolitan areas (Toronto, Ottawa, Hamilton, Sudbury, etc.) plus a catch-all for rural areas within each zone. Store this on the load record at creation time.
Confidence calculation: base = 0.5 + (min(load_count, 10) / 10 × 0.4) + recency_bonus. If most recent load was within 7 days, recency_bonus = 0.1. If within 30 days, recency_bonus = 0.05. If older, recency_bonus = 0.0.

Caching strategy: Store every DAT response in the rate_cache table with a source field of ‘dat’ and an expires_at timestamp set to 4–8 hours from fetch time. Before making an API call, check the cache first. Only call the API if cache is expired or missing for the lane.
Error handling: If DAT API returns an error (rate limit, auth failure, no data for lane), log the error, skip to next source in cascade. Never block a quote because DAT is down.

Before any API integrations are configured, the rate_cache table can be manually populated with rates looked up from the DAT website, industry contacts, or carrier negotiations. This is the MVP quoting data.

3.5 Source 5: AI-Powered Rate Estimation
When lane-specific data is unavailable from the sources above, an AI model can analyze the available context and produce a reasonable estimate. This is not hallucinated pricing — the AI receives structured data and applies reasoning.
How It Works
The TMS Settings page includes an AI Provider configuration with a selector (Claude API / OpenAI API) and an API key field. When the quoting engine reaches this layer in the cascade, it constructs a prompt with all available context and sends it to the configured AI provider.
Prompt Template

Respond ONLY with a JSON object. No explanation.
User: Estimate the carrier cost for this shipment:
Lane: {origin_city}, {origin_province} → {dest_city}, {dest_province}

- Current Canadian diesel price: {diesel_price_cad_per_litre}

  {list of 3-5 nearest cached lane rates with dates}
- Our historical average margin on similar lanes: {avg_margin}%
- Season: {season} (affects demand patterns)
- Day type: {weekday/weekend} (affects availability)
Respond with:
{
  "estimated_carrier_rate_per_mile": number,
  "estimated_total_carrier_cost": number,
  "confidence": number (0.0-1.0),
  "reasoning": "brief explanation",
  "rate_range_low": number,
  "rate_range_high": number
}
Implementation Notes
Model selection: Use Claude Sonnet 4.5 (claude-sonnet-4-5-20250929) for the best balance of speed, accuracy, and cost. Haiku 4.5 is faster but less nuanced for rate analysis. OpenAI’s GPT-4o is a viable alternative.
Cost control: Each AI quote call costs approximately $0.01–$0.03. At 50 quotes/day that’s under $1.50/day. Cache the AI response in rate_cache with a 24-hour TTL.
Failsafe: If the AI returns malformed JSON, unparseable response, or an API error, log it and fall through to the benchmark formula. Never let an AI failure block a quote.
Learning loop: When a load that was AI-quoted is completed, compare the actual carrier cost to the AI estimate. Store this delta. Over time, apply a correction factor to AI estimates for specific lane corridors.

4. Fuel Surcharge Module
The fuel surcharge is a variable component added to every quote. It accounts for fluctuations in diesel prices above a baseline. The module auto-updates weekly from public Canadian fuel price data.
4.1 Data Source
Natural Resources Canada (NRCan) publishes weekly average diesel prices for 45 Canadian cities through the Kalibrate Daily Price Survey. The Ontario government also publishes weekly diesel prices for 10 Ontario markets through their Open Data portal. Both are free and require no API key.
Primary: NRCan diesel price page at natural-resources.canada.ca/domestic-international-markets/diesel-prices. Updated weekly. Scrape or manually update each Monday.
Secondary: Ontario Open Data fuels price survey (open.canada.ca). CSV download with weekly diesel prices for Ontario markets. Can be automated via download script.
Tertiary: GlobalPetrolPrices.com offers a data API for Canadian diesel prices, updated weekly. Current Canadian average is approximately CAD $1.64/litre.
4.2 Fuel Surcharge Formula
// Canadian Trucking Standard Formula
base_diesel_price = 1.25   // CAD per litre (set at contract baseline)
current_diesel_price = 1.64 // CAD per litre (updated weekly)
fuel_consumption = 40       // litres per 100 km (avg tractor-trailer)
fuel_surcharge_per_km = (current_diesel_price - base_diesel_price)
                        × (fuel_consumption / 100)
// For a 644 km haul (Toronto to Sudbury):
fuel_surcharge = fuel_surcharge_per_km × 644
              = (1.64 - 1.25) × 0.40 × 644
              = $100.46
Alternatively, many Canadian brokers use the Freight Carriers Association of Canada (FCA) fuel index, which publishes a weekly fuel surcharge percentage. If the FCA index is available, applying their published percentage to the base rate is the simplest approach.
Database Storage
CREATE TABLE fuel_index (
  id              UUID PRIMARY KEY,
  source          VARCHAR(50),     -- 'nrcan', 'ontario_open_data', 'fca', 'manual'
  price_per_litre DECIMAL(6,4),    -- CAD
  effective_date  DATE,
  region          VARCHAR(100),    -- 'Ontario', 'National', city name
  created_at      TIMESTAMP DEFAULT NOW()
);
The quoting engine always uses the most recent fuel_index record to calculate the surcharge. A settings toggle allows the user to use the national average or a specific city’s price.
5. Quote Calculation Pipeline
This section defines the exact sequence of operations when a quote is requested. This is the core algorithm to implement.
Step 1: Distance Calculation
INPUT: origin_address, destination_address
// Geocode addresses to coordinates (if not already lat/lng)
origin_coords = mapbox_geocode(origin_address)
dest_coords = mapbox_geocode(destination_address)
// Get driving distance (road miles, not straight-line)
// Check distance_cache table first
cached = SELECT * FROM distance_cache
         WHERE origin_hash = hash(origin_coords)
           AND dest_hash = hash(dest_coords)
           AND created_at > NOW() - INTERVAL '30 days'
IF cached EXISTS:
  distance_km = cached.distance_km
  distance_miles = cached.distance_miles
  drive_time_hours = cached.drive_time_hours
ELSE:
  result = mapbox_directions_api(origin_coords, dest_coords)
  distance_km = result.distance / 1000
  distance_miles = distance_km × 0.621371
  drive_time_hours = result.duration / 3600
  INSERT INTO distance_cache (...)
Step 2: Region Normalization
// Map coordinates to named regions for rate cache lookup
origin_region = normalize_region(origin_coords)
dest_region = normalize_region(dest_coords)
// Region mapping logic:
// If within 50km of a major city center → use city name
// Major Ontario cities: Toronto, Ottawa, Hamilton, London,
//   Kitchener, Windsor, Sudbury, Thunder Bay, Barrie,
//   Kingston, Peterborough, Oshawa, Sault Ste Marie
// Otherwise → use province + 'Rural' (e.g., 'ON-Rural-North')
Step 3: Rate Lookup (Cascade)
function get_rate(origin_region, dest_region, equipment_type, pickup_date):
  // Source 1: Historical loads
  historical = query_historical_loads(origin_region, dest_region, equipment_type)
  IF historical.load_count >= 5:
    RETURN { rate: historical.avg_carrier_rate_per_mile,
             source: 'historical', confidence: calculate_confidence(historical) }
  // Source 2: DAT API
  IF settings.dat_api_key IS CONFIGURED:
    cached_dat = check_rate_cache('dat', origin_region, dest_region, equipment_type)
    IF cached_dat AND NOT expired:
      rate_data = cached_dat
    ELSE:
      TRY: rate_data = call_dat_api(origin, dest, equipment_type)
           store_in_cache(rate_data, source='dat', ttl=6_hours)
      CATCH: log_error, continue to next source
    IF rate_data:
      // Blend with historical if partial historical data exists
      IF historical.load_count >= 1:
        blended = (historical.avg × 0.3) + (rate_data.avg × 0.7)
        RETURN { rate: blended, source: 'dat+historical', confidence: 0.85 }
      RETURN { rate: rate_data.avg_rate_per_mile, source: 'dat', confidence: 0.80 }
  // Source 3: Truckstop API
  IF settings.truckstop_api_key IS CONFIGURED:
    // Same pattern as DAT: check cache, call API, fallback
    ...
  // Source 4: Manual rate cache
  manual = check_rate_cache('manual', origin_region, dest_region, equipment_type)
  IF manual:
    age_days = days_since(manual.fetched_at)
    conf = max(0.3, 0.7 - (age_days / 60 × 0.4))  // Degrades over 60 days
    RETURN { rate: manual.rate_per_mile, source: 'manual_cache', confidence: conf }
  // Source 5: AI estimation
  IF settings.ai_api_key IS CONFIGURED:
    TRY:
      context = gather_nearby_lanes(origin_region, dest_region)
      ai_result = call_ai_rate_estimate(lane_details, context)
      store_in_cache(ai_result, source='ai', ttl=24_hours)
      RETURN { rate: ai_result.rate_per_mile, source: 'ai', confidence: 0.55 }
    CATCH: log_error, continue
  // Source 6: Benchmark formula (always works)
  benchmark = benchmark_table[distance_band][equipment_type]
  seasonal = get_seasonal_factor(pickup_date)
  RETURN { rate: benchmark.midpoint × seasonal, source: 'benchmark', confidence: 0.30 }
Step 4: Fuel Surcharge
fuel = get_latest_fuel_index()
fuel_surcharge = (fuel.price_per_litre - BASE_DIESEL_PRICE)
                × FUEL_CONSUMPTION_PER_100KM / 100
                × distance_km
Step 5: Margin Application
// Margin is percentage of REVENUE, not cost
// shipper_rate = carrier_cost / (1 - margin)
carrier_cost = (rate_per_mile × distance_miles) + fuel_surcharge
// Determine margin percentage
margin = get_margin(shipper_id, origin_region, dest_region, confidence)
// Margin logic:
// Default margin: 15%
// New shipper (first load): 10-12% (win the business)
// Repeat shipper, 10+ loads: 15-18% (earned trust)
// Hot freight / urgent: 20-25% (premium)
// Low confidence quote: +2-3% buffer (hedge uncertainty)
// Override: user can manually adjust on the quote form
shipper_rate = carrier_cost / (1 - margin)
margin_dollars = shipper_rate - carrier_cost
Step 6: Assemble and Store Quote
quote = {
  id: generate_uuid(),
  reference: generate_quote_ref(),   // e.g., MYR-Q-2026-0042
  shipper_id: shipper_id,
  origin_address, origin_lat, origin_lng, origin_region,
  dest_address, dest_lat, dest_lng, dest_region,
  equipment_type, weight_lbs,
  pickup_date,
  distance_miles, distance_km, drive_time_hours,
  
  // Pricing
  rate_per_mile,
  carrier_cost_estimate: carrier_cost,
  fuel_surcharge,
  shipper_rate,
  margin_percent: margin,
  margin_dollars,
  
  // Source & confidence
  rate_source,              // 'historical', 'dat', 'truckstop', etc.
  rate_source_detail,       // JSON with raw source data
  confidence,               // 0.0 - 1.0
  confidence_label,         // 'HIGH', 'MEDIUM', 'LOW'
  
  // Lifecycle
  status: 'draft',          // draft, sent, accepted, declined, expired
  valid_until: pickup_date, // Quote expiry
  created_at: now(),
  
  // Feedback (filled after load completes)
  actual_carrier_cost: null,
  quote_accuracy: null      // Computed: 1 - abs(estimated - actual) / actual
}
INSERT INTO quotes (...)
RETURN quote

9. Learning Feedback Loop
The quoting engine gets smarter with every completed load. This section defines how actual transaction data feeds back into the system.
Automatic Feedback
When a load that originated from a quote transitions to “delivered” status, the system automatically:
Computes quote accuracy: accuracy = 1 - abs(estimated_carrier_cost - actual_carrier_cost) / actual_carrier_cost.
Stores the accuracy on the quote record.
Updates the aggregate accuracy metrics per source (e.g., DAT quotes average 94% accuracy, AI quotes average 87%).
The actual carrier cost becomes a historical data point for future quotes on this lane.
Correction Factors
Over time, if a specific data source consistently over- or under-estimates on certain corridors, the system applies a correction factor:
// If DAT consistently underestimates Toronto-Sudbury by 5%:
correction_factor = avg(actual_cost / estimated_cost) for lane over last 20 loads
adjusted_rate = dat_rate × correction_factor
This is stored per source + lane combination in a quote_corrections table and applied silently during rate lookup.
Reporting Dashboard
The TMS should include a Quote Analytics section showing: average quote accuracy by source, accuracy trend over time, most-quoted lanes, win rate (quotes that became booked loads), margin realization (actual vs. quoted margin), and source utilization (percentage of quotes from each cascade level).
